# الربط: التطبيق ← Railway ← المستودع

## الصورة الكاملة

```
المستودع (GitHub)          Railway                    التطبيق (على الجوال)
─────────────────          ───────                    ─────────────────────
يحتوي على:                 يسحب من المستودع           يحمّله المستخدم
• ملفات السيرفر (API)       ويشغّل السيرفر              (Expo Go أو بناء APK)
• أو ملفات التطبيق          (تسجيل، دخول، قاعدة        ويتصل بريلوي عبر
  + مجلد الـ API              البيانات)                  BASE_URL
```

---

## ١) ريلوي ياخذ من المستودع (العب، المنطق، الطريقة، كل شيء)

ريلوي يشغّل **سيرفر فقط** (لا يشغّل تطبيق الموبايل). لذلك اللي ريلوي ياخذه من المستودع هو **ملفات السيرفر**:

- يجب أن المستودع المربوط بريلوي فيه على الأقل:
  - **api_server.py** — تسجيل، دخول، بروفايل
  - **database.py** — اتصال قاعدة البيانات
  - **requirements.txt**
  - **Procfile** — أمر التشغيل

هذي = "المنطق والطريقة" اللي على السيرفر (حسابات، قاعدة بيانات).  
اللعب الفعلي (الورق، الدور، البوت) يعمل **داخل التطبيق على الجوال**؛ السيرفر يوفّر الحساب والربط بقاعدة البيانات.

### إذا المستودع فيه تطبيق الموبايل + مجلد الـ API

- في إعدادات الخدمة على Railway:
  - **Root Directory:** ضعه `uno-app-api` (أو اسم المجلد اللي فيه api_server.py و database.py).
- بهذا ريلوي يبني ويشغّل **الـ API فقط** من المستودع، وياخذ كل ملفات السيرفر من هناك.

### إذا المستودع = الـ API فقط

- لا تحتاج Root Directory؛ ريلوي يشغّل من جذر المستودع.
- تأكد أن الجذر فيه: api_server.py, database.py, requirements.txt, Procfile.

---

## ٢) التطبيق مرتبط بريلوي

- التطبيق (اللي على الجوال) يتصل بريلوي عبر **رابط السيرفر**.
- في مشروع التطبيق الملف **src/api.js** فيه:
  ```js
  const BASE_URL = 'https://web-production-554dd.up.railway.app';
  ```
- هذا = "التطبيق مرتبط بريلوي". أي تسجيل أو دخول من التطبيق يروح لريلوي، وريلوي يستخدم قاعدة البيانات اللي مربوطة فيه.

---

## ٣) خلاصة

| المكوّن | المصدر | الدور |
|---------|--------|--------|
| **المستودع** | GitHub | يحوي كود السيرفر (والـ API) و/أو كود التطبيق |
| **ريلوي** | يسحب من المستودع | يشغّل السيرفر (API) وياخذ منه المنطق والطريقة وكل شيء خاص بالسيرفر |
| **التطبيق (موبايل)** | يُبنى من كود التطبيق (نفس المستودع أو مستودع آخر) | يعمل على الجوال ومرتبط بريلوي عبر BASE_URL |

بهذا: التطبيق مرتبط بريلوي، وريلوي ياخذ من المستودع كل شيء خاص بالسيرفر (العب والمنطق والطريقة على مستوى السيرفر).
