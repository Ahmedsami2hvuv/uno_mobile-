# ثلاثة أشياء مهمة

## ١) تطبيق APK تثبته بالهاتف

**نعم، تقدر تسوي APK** وتثبته على الهاتف مثل أي تطبيق.

الطريقة (بعد ما يكون المشروع جاهز على GitHub و Expo):

1. سجّل حساب على [expo.dev](https://expo.dev).
2. من جهازك (أو من جهاز يبني المشروع) داخل مجلد التطبيق:
   ```bash
   npx eas-cli login
   npx eas build --platform android --profile preview
   ```
3. بعد انتهاء البناء يعطيك Expo **رابط تحميل APK**. حمّله وثبّته على الهاتف.

الـ APK = نفس التطبيق اللي يشتغل مع Expo Go، لكنه يعمل **بدون** Expo Go ومرتبط بريلوي عبر ملف الربط.

---

## ٢) ملف يربط التطبيق بريلوي (مثل توكن البوت)

البوت عنده **توكن** في ملف أو في الإعدادات؛ التطبيق عنده **ملف ربط واحد**:

- **اسم الملف:** `src/railwayConfig.js`
- **فيه فقط:** رابط ريلوي (الدومين العام لسيرفر الـ API).

مثال محتواه:
```js
export const RAILWAY_APP_URL = 'https://web-production-554dd.up.railway.app';
```

هذا الملف = **ربط التطبيق بريلوي**. لو غيّرت الرابط هنا فقط، كل التطبيق يروح على الرابط الجديد. ما تحتاج تعدّل في كل الملفات.

---

## ٣) ريلوي ياخذ معلومات اللعب من GitHub

- ريلوي **مربوط بمستودعك على GitHub**.
- كل ما تدفع (Push) تعديلات على المستودع، ريلوي يقدر يعيد البناء ويشغّل النسخة الجديدة.
- السيرفر اللي على ريلوي ياخذ **معلومات اللعب والحسابات** من الكود اللي في المستودع (ملفات مثل api_server.py، database.py، إلخ).

يعني: **المستودع = مصدر المنطق والبيانات**، و**ريلوي = يشغّل من المستودع**.

---

## ملخص

| المطلوب | الحل |
|---------|------|
| APK تثبته بالهاتف | بناء عبر `eas build --platform android` ورابط التحميل من Expo |
| ملف يربط التطبيق بريلوي (مثل توكن البوت) | `src/railwayConfig.js` فيه `RAILWAY_APP_URL` |
| ريلوي ياخذ معلومات اللعب من GitHub | ريلوي مربوط بالمستودع ويشغّل السيرفر من الكود على GitHub |
