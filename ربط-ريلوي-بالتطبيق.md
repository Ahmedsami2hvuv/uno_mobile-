# كيف تربط Railway بالتطبيق (خطوة بخطوة)

## الفكرة باختصار

- **Railway** يوفّر السيرفر (البوت + قاعدة البيانات). مشروع **uno** عندك عليه بالفعل «2/2 services online».
- **التطبيق** (uno-mobile) يعمل على هاتفك ولا يُرفع على Railway. المطلوب فقط أن يعرف **عنوان الـ API** على Railway ليرسل له طلبات تسجيل/دخول/بروفايل.
- بعد إضافة **سيرفر الـ API** (الملف api_server.py) لمشروع uno على Railway، تحصل على **رابط (URL)**. هذا الرابط تضعه في التطبيق في ملف **src/api.js** في سطر **BASE_URL**.

---

## الجزء ١ — إضافة الـ API لمشروع uno على Railway

### 1) إضافة الملف في مستودع البوت (GitHub)

1. افتح مستودع **uno** على GitHub: https://github.com/Ahmedsami2hvuv/uno  
2. من مشروعك المحلي **uno-mobile** انسخ الملف:
   - من: `uno-mobile\uno-api\api_server.py`
   - إلى: داخل مستودع **uno** (نفس مستوى `bot.py`).
3. أضف في مستودع uno ملف **requirements.txt** سطراً إن لم يكن موجوداً:
   ```
   fastapi
   uvicorn
   ```
4. ارفع التعديلات (Commit + Push) إلى GitHub.

### 2) على Railway — إضافة خدمة (Service) للـ API

1. ادخل إلى **railway.com** وافتح مشروع **uno** (اللي يظهر عندك 2/2 services).
2. اضغط على المشروع **uno** لفتحه.
3. اضغط زر **"+ New"** (أو Add Service) داخل المشروع.
4. اختر **"GitHub Repo"** واختر المستودع **uno** نفسه (نفس المستودع اللي مربوط حالياً).
5. بعد إنشاء الخدمة الجديدة:
   - اضغط على الخدمة الجديدة (مثلاً اسمها "uno-api" أو غيره).
   - ادخل إلى **Settings** أو **Variables**.
   - في **Build Command** اتركه فارغاً أو: `pip install -r requirements.txt`
   - في **Start Command** أو **Start** ضع:
     ```
     uvicorn api_server:app --host 0.0.0.0 --port $PORT
     ```
   - أضف **نفس متغيرات البيئة** اللي عند الخدمة الأولى (مثلاً `DATABASE_URL`) حتى الـ API يتصل بنفس قاعدة البيانات. في Railway عادة تضغط **Variables** وتنسخ من الخدمة الأخرى أو تربط بنفس Database.
6. من تبويب **Settings** للخدمة الجديدة، ابحث عن **"Generate Domain"** أو **"Public Networking"** وولّد دومين (Domain). سيظهر لك رابط مثل:
   ```
   https://uno-api-production-xxxx.up.railway.app
   ```
7. انسخ هذا الرابط — سنستخدمه في التطبيق.

---

## الجزء ٢ — ربط التطبيق (uno-mobile) بـ Railway

1. على جهازك افتح مشروع **uno-mobile**.
2. افتح الملف **src/api.js**.
3. غيّر السطر:
   ```js
   const BASE_URL = 'https://YOUR-SERVER.repl.co';
   ```
   إلى الرابط اللي نسخته من Railway، مثلاً:
   ```js
   const BASE_URL = 'https://uno-api-production-xxxx.up.railway.app';
   ```
4. احفظ الملف.
5. شغّل التطبيق (Run-Uno) وافتح **حسابي** ثم **تسجيل / دخول**. عندها التطبيق سيرسل الطلبات إلى Railway ويستخدم نفس قاعدة البيانات.

---

## ملخص

| أين | ماذا تفعل |
|-----|-----------|
| **GitHub (مستودع uno)** | تضيف `api_server.py` و `fastapi` و `uvicorn` في المتطلبات وتدفع التعديلات. |
| **Railway (مشروع uno)** | تضيف خدمة جديدة من نفس المستودع، Start Command: `uvicorn api_server:app --host 0.0.0.0 --port $PORT`، وتولّد دومين. |
| **التطبيق (uno-mobile)** | في `src/api.js` تضع رابط الدومين في **BASE_URL**. |

بعدها التطبيق «مرتبط» بريلوي: أي تسجيل أو دخول من التطبيق يمر عبر السيرفر على Railway ويستخدم نفس قاعدة البيانات مع البوت.
