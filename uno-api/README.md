# API التطبيق — ربط مع بوت أونو وقاعدة البيانات

هذا المجلد يحتوي على **سيرفر API** يُضاف إلى مستودع البوت [Ahmedsami2hvuv/uno](https://github.com/Ahmedsami2hvuv/uno) حتى يتصل التطبيق بنفس قاعدة البيانات والحساب.

## ما الذي يفعله الـ API؟

- **POST /api/auth/register** — تسجيل مستخدم جديد (من التطبيق)، يُخزَن في جدول `users` نفسه.
- **POST /api/auth/login** — دخول بيوزر نيم وكلمة السر (نفس حساب البوت).
- **GET /api/profile** — جلب البروفايل (مع توكن الدخول).

المستخدمون المسجّلون من التطبيق يُعطَوْن `user_id` سالباً حتى لا يتعارضوا مع معرّفات التليجرام.

## كيف أضيفه إلى مستودع البوت؟

1. انسخ الملف **`api_server.py`** إلى **جذر مستودع uno** (نفس مستوى `bot.py` و `database.py`).

2. ثبّت المتطلبات:
   ```bash
   pip install fastapi uvicorn
   ```

3. شغّل السيرفر (مع البوت أو بجانبه):
   ```bash
   uvicorn api_server:app --host 0.0.0.0 --port 8000
   ```

4. على Repl.it أو Railway: أضف في **Procfile** سطراً ثانياً، مثلاً:
   ```
   web: uvicorn api_server:app --host 0.0.0.0 --port $PORT
   ```
   أو شغّل العملية الثانية يدوياً حسب منصتك.

5. في التطبيق ضع **عنوان السيرفر** في الإعدادات (مثلاً `https://your-app.repl.co` أو `https://your-api.railway.app`).

## ملاحظة

- جدول `users` في البوت يستخدم `password_key` كنص (كلمة السر كما هي). الـ API يخزّن ويقارن بنفس الطريقة ليتوافق الدخول من البوت والتطبيق.
- للتطبيق فقط: المستخدمون الجدد يأخذون `user_id` سالباً (مثل -1, -2, ...) حتى لا يتداخلوا مع `user_id` التليجرام.
