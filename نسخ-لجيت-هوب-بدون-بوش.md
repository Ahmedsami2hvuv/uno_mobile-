# كيف تضع التعديلات على GitHub بدون Push من حاسوبك

## الفكرة

- **الحاسوب:** التعديلات اللي أنا (المساعد) أعدّلها تظهر في الملفات داخل مجلد المشروع عندك (uno-mobile).
- **GitHub:** تريد أن تكون النسخة النهائية هناك، **بدون** أن تفتح Terminal وتكتب `git push`.

## الطريقة: تعديل الملفات مباشرة على موقع GitHub

1. افتح مستودعك على GitHub (مثلاً `github.com/اسمك/uno-mobile` أو أي مستودع تستخدمه للتطبيق).
2. ادخل إلى الملف الذي تريد تحديثه (مثلاً `src/api.js`).
3. اضغط على أيقونة **القلم (Edit)** أعلى الملف.
4. **احذف** المحتوى القديم و**الصق** المحتوى الجديد من الملف نفسه على حاسوبك (من مجلد uno-mobile).
   - أو افتح الملف على حاسوبك من Cursor/المحرر، انسخ كل المحتوى، ثم الصقه في محرر GitHub.
5. في الأسفل اكتب رسالة وصف (مثلاً: تحديث api وربط Railway) ثم اضغط **Commit changes**.

بهذا الشكل التعديلات تصير على GitHub **بدون** استخدام أمر push من الحاسوب.

## إذا أردت رفع ملف جديد (لم يكن في المستودع)

1. من صفحة المستودع على GitHub اضغط **Add file** → **Create new file**.
2. اكتب اسم الملف (مثلاً `src/api.js` مع المسار).
3. الصق المحتوى من الملف من حاسوبك.
4. **Commit** ثم احفظ.

## ملخص

| أين التعديل؟ | أين تريده؟ | ماذا تفعل؟ |
|--------------|------------|------------|
| في الملفات على حاسوبك (بعد ما أعدّل أنا) | على GitHub | افتح الملف على GitHub → Edit → الصق محتوى الملف من حاسوبك → Commit |

كل الشغل يبقى على GitHub لأنك أنت تنقل المحتوى يدوياً (نسخ ولصق) من الملفات المحلية إلى محرر GitHub. لا تحتاج أوامر push إذا عملت بهذه الطريقة.
